Known Problems

1. Missing libXm

1. Cannot run from laptop if not connected to a network

1. Missing libXm Some users have reported that on their system the RH 6.0 and SuSE 6.2 version of AIPS++ cannot be run either from the CDROM or from disk. The reason is that this version was linked against a shared LessTif library libXm.so. Hence AIPS++ will not run from the CDROM or from disk uncless you have Motif (a commercially available product) or lesstif (available under the GPL) installed.

If you do not have lesstif installed, the easiest fix is to install the library from the following rpms lesstif-0.85-1.i386.rpm and lesstif-devel-0.85-1.i386.rpm.

These rpms are widely available. A copy is available on our server aips2.nrao.edu. To install, do the following as root:

ftp aips2.nrao.edu

binary

cd pub/misc

get lesstif-0.85-1.i386.rpm lesstif-devel-0.85-1.i386.rpm

quit

rpm -i lesstif-0.85-1.i386.rpm lesstif-devel-0.85-1.i386.rpm

AIPS++ should then run correctly. You should not have to reinstall AIPS++ if you have already done so.

2. Cannot run from laptop if not connected to a network.

This is a known defect. A workaround is to change the permissions of the keys file in the AIPS++ home directory. In the directory where AIPS++ is installed:

rm -rf keys

mkdir keys

chmod a-rwx keys
